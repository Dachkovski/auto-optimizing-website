# Data Traceability & Memory Architecture

The Darwin Engine is built on a highly traceable data architecture. Every interaction, visual state, and AI decision is strictly logged and can be reconstructed. This ensures we can audit exactly **who** saw **what**, **when**, and **why** the AI made specific decisions.

## 1. User & Session Traceability
Every event in the system is tied to:
- \`visitorId\`: A unique, persistent identifier for the user (stored in localStorage).
- \`sessionId\`: A short-lived identifier for the current browsing session.

## 2. Interaction Telemetry (The "What" and "How")
When a user interacts with a variant, the telemetry bridge (\`window.darwin.trackEvent\`) captures granular data:
- **Click Streams**: \`interaction_click\` events store the exact DOM text clicked.
- **Form States**: Input values at the moment of interaction are saved in \`metadataJson.formState\`.
- **3D Context**: Custom JS logic can update \`window.darwin.sceneState\`, which is saved upon click (e.g., "User is looking at the red sphere").
- **Visual Memory**: On session bounce, the start and end visual states are captured as Base64 screenshots. These are analyzed by a vision LLM (GPT-5.5), and the resulting text insight is stored as a \`visual_analysis\` event. The physical screenshots are persisted to \`public/snapshots/\` and linked in the DB.

## 3. AI Thought Process (The "Why")
To trace the AI's decision-making process:
- **Research Logs (\`researchLogs\` table)**: The Analyzer LLM evaluates the telemetry and generates an \`observation\` (what went wrong/right) and a \`hypothesis\` (what to do next).
- **Generation Prompt (\`variants.generationPrompt\`)**: When the Evolver LLM generates a new variant, the **exact, literal prompt** it received (including the concatenated \`userHistoryContext\`, the hypothesis, and the creative directive) is saved directly into the \`variants\` table under the \`generationPrompt\` column. 
- **Code Output (\`variants.contentJson\`)**: The exact HTML/CSS/JS output generated by the LLM is saved and immutably bound to that generation ID.

## 4. Reconstructing a Journey
Because of this architecture, the Admin Dashboard can reconstruct a complete, chronological "Live User Journey" for any visitor:
1. What the AI measured (\`observation\`).
2. What the AI planned (\`hypothesis\`).
3. The exact prompt the AI was fed (\`generationPrompt\`).
4. The exact code the AI built (\`contentJson\`).
5. What the user did on that code (\`interaction_click\`, \`formState\`, \`sceneState\`).
6. What the UI looked like at the end (\`startImage\` / \`latestImage\`).
